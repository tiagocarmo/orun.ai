# Plan: Fix agent-delegation skill and documentation

## Problem

When starting a NEW conversation (no context), the agent calls `actor` tool with invalid arguments:

1. **`operation` passed as string** → `"expected object, received string"`
2. **`timeout_ms` at root level** → `"unrecognized_keys: timeout_ms"`

Current documentation (SKILL.md, AGENTS.md, CLAUDE.md) already has rules about this, but the agent still makes the same errors. Root cause analysis:

- **CLAUDE.md examples use pseudo-code** (`actor({...}`) instead of the actual JSON the tool expects (`{"operation": {...}}`). The agent conflates JS function call syntax with the JSON payload.
- **The tool's JSON schema is strict**: top-level keys are ONLY `operation`. Fields like `timeout_ms`, `context`, `model` at root are rejected. These must go INSIDE `operation`.
- **SKILL.md examples lack the "wrong vs right" contrast** at the very top — the first thing an agent sees should be an unmistakable comparison.
- **AGENTS.md has too much prose** between the examples and the schema, making it easy for the agent to drift.

## Approach

Update 3 files with a unified, unambiguous format:

1. **`.agents/skills/agent-delegation/SKILL.md`** — Rewrite with:
   - Absolute top: a JSON-only "copy-paste" reference (no pseudo-code)
   - Explicit "NEVER do this / ALWAYS do this" with real JSON payloads
   - Single "Golden Rule" box that's impossible to miss
   - Remove all JS-style pseudo-code examples (`actor({...}`)
   - Add `workflow` tool documentation (currently uses flat `operation` string — the EXCEPTION)

2. **`CLAUDE.md`** — Replace pseudo-code examples with actual JSON payloads:
   - Show `{"operation": {...}}` format, not `actor({...})`
   - Add explicit schema reminder: "top-level key is ONLY `operation`"
   - Keep the checklist and error handling sections

3. **`AGENTS.md`** — Simplify the "Contrato de Invocação" section:
   - Remove redundant prose that duplicates SKILL.md
   - Keep the JSON examples (already correct)
   - Add cross-reference: "Full reference: `.agents/skills/agent-delegation/SKILL.md`"
