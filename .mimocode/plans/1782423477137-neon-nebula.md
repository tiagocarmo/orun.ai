# Plano: Botão de Ajuda Global nas Páginas do Sistema

## Objetivo

Adicionar um botão de ajuda (?) no topo-right de todas as páginas do dashboard que abre um modal com informações sobre a tela atual. O conteúdo deve ser fácil de manter.

## Arquitetura

```
Topbar (já existe)
  └─ HelpButton (novo)
       └─ HelpModal (novo, reutiliza Modal existente)
            └─ help-content.ts (registry centralizado de conteúdo por rota)
```

## Arquivos a Criar/Modificar

### 1. `src/components/ui/help-modal.tsx` (NOVO)

Componente `HelpButton` + `HelpModal`:
- Botão `?` (ícone SVG circle-question) posicionado no topbar
- Abre modal com título e conteúdo da página atual
- Usa o `Modal` existente como base (simplificado: sem botão de confirmação)
- Props: `title: string`, `content: React.ReactNode`

### 2. `src/lib/help-content.ts` (NOVO)

Registry centralizado mapeando rotas → conteúdo de ajuda:

```typescript
interface HelpContent {
  title: string;
  content: React.ReactNode;
}

export const helpContentByRoute: Record<string, HelpContent> = {
  "/": { title: "Painel", content: "..." },
  "/agents": { title: "Agentes", content: "..." },
  "/leads": { title: "Leads", content: "..." },
  // ... todas as rotas
};
```

Cada entrada contém:
- `title`: Nome da tela
- `content`: JSX com seções "O que é" e "Como usar" (3-5 bullets cada)
- Conteúdo moderado: informativo o suficiente para guiar o usuário, sem sobrecarregar
- **Organização: centralizada em um único arquivo** para facilitar revisão e manutenção

### 3. `src/components/layout/topbar.tsx` (MODIFICAR)

Adicionar `HelpButton` ao lado direito do topbar, antes do sino de notificações.

### 4. `src/components/ui/modal.tsx` (MODIFICAR)

Tornar `onConfirm` opcional (já é, mas garantir que o footer só aparece quando há `onConfirm`). Ou criar uma versão simplificada para o help.

### 5. `CLAUDE.md` (MODIFICAR)

Adicionar regra: ao criar/modificar uma tela, incluir ou atualizar o conteúdo de ajuda em `src/lib/help-content.ts`.

### 6. `AGENTS.md` (MODIFICAR)

Adicionar nota sobre manutenção do conteúdo de ajuda.

## Implementação Detalhada

### help-content.ts

```typescript
import React from "react";

interface HelpContent {
  title: string;
  content: React.ReactNode;
}

export const helpContentByRoute: Record<string, HelpContent> = {
  "/": {
    title: "Painel",
    content: React.createElement("div", { className: "space-y-3" },
      React.createElement("h3", { className: "font-medium text-ink" }, "O que é"),
      React.createElement("p", null, "Visão geral do sistema com métricas principais."),
      React.createElement("h3", { className: "font-medium text-ink" }, "Como usar"),
      React.createElement("ul", { className: "list-disc list-inside space-y-1" },
        React.createElement("li", null, "Cards mostram totais de leads, qualificações e execuções"),
        React.createElement("li", null, "Gráficos mostram evolução de leads e performance de agentes"),
      ),
    ),
  },
  // ... outras rotas
};
```

### help-modal.tsx

Componente client que:
1. Obtém pathname via `usePathname()`
2. Busca conteúdo no registry
3. Renderiza botão `?` que abre modal
4. Se não houver conteúdo para a rota, esconde o botão

### topbar.tsx

Adicionar import e renderização do `HelpButton` na div de botões à direita.
