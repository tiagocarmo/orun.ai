# Orun.AI Workforce Platform — MVP Task Breakdown

## Overview

Decomposição do projeto Orun.AI em **28 tarefas** agrupadas em **3 workstreams paralelos**, projetados para zero conflito de arquivos em git worktrees.

## Workstream Strategy

| Workstream | Escopo | Paths Exclusivos |
|---|---|---|
| **WS-A: Foundation** | Config, DB, tipos, utilitários, LLM | `package.json`, `prisma/*`, `src/lib/*`, `.env*` |
| **WS-B: Backend** | Agentes, engine, server actions, API | `src/lib/agents/**`, `src/lib/ai/**`, `src/lib/mcp/**`, `src/app/api/**`, `src/app/actions/**` |
| **WS-C: UI** | Layout, componentes, páginas | `src/app/layout.tsx`, `src/app/page.tsx`, `src/app/(dashboard)/**`, `src/components/**` |

**Merge order**: WS-A → WS-B + WS-C (paralelo) → Integração

## Merge Order

```
1. WS-A (Foundation) — PRIMEIRO
   └── Fornece: schema, types, db client, utils, LLM client

2. WS-B + WS-C (paralelo, após merge de WS-A)
   ├── WS-B: backend usa types/db de WS-A
   └── WS-C: UI usa layout de T-A1 + componentes próprios

3. Integração
   └── Wire server actions (WS-B) → páginas (WS-C)
   └── Full build + test
   └── Versão 0.1.0
```

## Test Strategy

| Nível | Ferramenta | Cobertura |
|---|---|---|
| Unit | Vitest | `src/lib/**`, validators, utils |
| Component | Vitest + @testing-library/react | `src/components/**` |
| Integration | Vitest | Server actions + mock Prisma |
| E2E | Playwright (opcional) | Fluxo: criar lead → qualificar → ver histórico |

## Verification (end-to-end)

1. `npm run build` — zero errors
2. `npm run test` — todos passam
3. UI: criar agente → criar lead → executar qualificação → ver run no histórico
4. Webhook: `curl -X POST /api/webhooks/leads` cria lead
5. Dashboard: métricas atualizam
6. Conversas: mensagens visíveis na thread
