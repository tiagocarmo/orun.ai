## O que PRECISA ser corrigido

### 1. Sidebar Fixa em Desktop (CRITICO)

**Problema:** Sidebar usa `lg:static` — fica no fluxo normal e scrolla com o conteudo.

**Solucao:** Mudar para `lg:fixed` com `h-screen` e ajustar o main content com `lg:ml-64`.

**Arquivos:** `src/components/layout/sidebar.tsx`, `src/components/layout/dashboard-shell.tsx`

**CSS atual (sidebar):**
```
fixed top-0 left-0 z-50 h-full w-64 ... lg:translate-x-0 lg:static lg:z-auto
```

**CSS correto (sidebar):**
```
fixed top-0 left-0 z-50 h-full w-64 ... lg:translate-x-0 lg:fixed
```

**CSS atual (dashboard-shell main):**
```
<div className="flex min-h-screen bg-canvas">
```

**CSS correto (dashboard-shell):**
```
<div className="flex h-screen bg-canvas">
  <Sidebar />
  <div className="flex-1 flex flex-col min-w-0 lg:ml-64">
    ...
  </div>
</div>
```

### 2. Busca de Leads no Run Agent

**Problema:** Input e um textarea JSON bruto. Usuario quer buscar leads existentes.

**Solucao:** 
- Adicionar `searchLeads(query)` em `src/app/actions/leads.ts`
- Substituir textarea por campo de busca com autocomplete
- Debounce 3s, min 3 letras, max 3 resultados
- Ao selecionar: preencher dados do lead automaticamente

**Arquivos:** `src/app/actions/leads.ts`, `src/app/(dashboard)/runs/page.tsx`

### 3. Contexto Enriquecido nas Conversas

**Problema:** Pagina de conversas mostra dados minimos.

**Solucao:** Enriquecer com:
- Nome, email, empresa do lead
- Ultima mensagem truncada com timestamp
- Status da conversa
- Quantidade de mensagens

**Arquivos:** `src/app/(dashboard)/conversations/page.tsx`, `src/components/conversations/conversation-list.tsx`
