## Problemas Identificados (Análise)

### 1. UX do Select de Lead (Run Agents)
- **Arquivo:** `src/app/(dashboard)/runs/page.tsx:141-170`
- **Problema:** Dropdown de busca de lead aparece com `position: absolute` mas o container não tem `position: relative`, causando posicionamento incorreto. O dropdown flutua acima do campo de input.
- **Solução:** Adicionar `relative` no container do input de busca para que o dropdown fique posicionado corretamente abaixo do campo.

### 2. Campo JSON Visível (Run Agents)
- **Arquivo:** `src/app/(dashboard)/runs/page.tsx:183-191`
- **Problemo:** Após selecionar um lead, o campo textarea "Entrada (texto ou JSON)" é preenchido automaticamente e fica visível, poluindo a UI.
- **Solução:** Ocultar o textarea quando um lead estiver selecionado. Manter o estado `input` internamente via state. Quando não houver lead selecionado, mostrar o textarea para entrada manual.

### 3. Botão Execute Habilitado sem Campos Obrigatórios
- **Arquivo:** `src/app/(dashboard)/runs/page.tsx:194-200`
- **Problema:** Botão "Executar Agente" usa `disabled={!agentSlug || !input.trim()}` mas deveria considerar que o lead também é obrigatório (quando selecionado, os dados devem vir do state interno).
- **Solução:** Lógica de habilitação: `disabled={!agentSlug || (!input.trim() && !selectedLead)}` - habilitar quando agente selecionado E (input manual preenchido OU lead selecionado).

### 4. Erro "Agent 'qualification' not found in registry"
- **Arquivo raiz:** `src/lib/agents/engine.ts:2` importa de `./registry` direto
- **Problema:** `engine.ts` importa `agentRegistry` de `./registry` (map vazio), mas o registro dos agentes acontece em `./index.ts` que nunca é importado por ninguém.
- **Solução:** Alterar `engine.ts:2` para importar de `"./index"` em vez de `"./registry"`. Isso garante que a barrel file execute o registro dos agentes antes do engine usar o registry.

### 5. Botão Arquivar Conversations
- **Arquivos:** `src/components/conversations/conversation-list.tsx`, `src/app/actions/conversations.ts`
- **Problema:** Status "archived" existe no schema mas não há server action nem UI para arquivar.
- **Solução:**
  - Criar `updateConversation` server action em `src/app/actions/conversations.ts`
  - Adicionar `updateConversationSchema` em `src/lib/validators.ts`
  - Criar componente `conversation-actions.tsx` seguindo o padrão de `lead-actions.tsx`
  - Adicionar botão de arquivar na lista de conversas e na página de detalhe
